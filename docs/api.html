API Client¶

Commitment options.

Solana nodes choose which bank state to query based on a commitment requirement set by the client. Clients may specify the following commitments:

solana.rpc.commitment.Commitment(x)¶

Type for commitment.

solana.rpc.commitment.Max = 'max'¶

The node will query the most recent bank confirmed by the cluster as having reached MAX_LOCKOUT_HISTORY confirmations.

solana.rpc.commitment.Recent = 'recent'¶

The node will query its most recent bank.

solana.rpc.commitment.Root = 'root'¶

The node will query the most recent bank having reached MAX_LOCKOUT_HISTORY confirmations on this node.

solana.rpc.commitment.Single = 'single'¶

The node will query the most recent bank having reached 1 confirmation.

API client to interact with the Solana JSON RPC Endpoint.

get_account_info(pubkey: Union[solana.publickey.PublicKey, str], commitment: NewType.<locals>.new_type = 'max', encoding: str = 'base64', data_slice: Optional[solana.rpc.types.DataSliceOpts] = None) → solana.rpc.types.RPCResponse[source]¶

Returns all the account info for the specified public key.

Parameters

  • pubkey – Pubkey of account to query, as base-58 encoded string or PublicKey object.

  • commitment – Bank state to query. It can be either “max”, “root”, “single” or “recent”.

  • encoding –

    (optional) Encoding for Account data, either “base58” (slow), “base64”, or “jsonParsed”. Default is “base64”.

    • ”base58” is limited to Account data of less than 128 bytes.

    • ”base64” will return base64 encoded data for Account data of any size.

    • ”jsonPrased” encoding attempts to use program-specific state parsers to return more human-readable and explicit account state data.

    If jsonParsed is requested but a parser cannot be found, the field falls back to base64 encoding, detectable when the data field is type. (jsonParsed encoding is UNSTABLE).

  • data_slice – (optional) Option to limit the returned account data using the provided offset: <usize> and length: <usize> fields; only available for “base58” or “base64” encoding.

>>> from solana.publickey import PublicKey
>>> solana_client = Client("http://localhost:8899")
>>> solana_client.get_account_info(PublicKey(1)) 
{'jsonrpc': '2.0',
 'result': {'context': {'slot': 33265073},
  'value': {'data': '',
   'executable': False,
   'lamports': 4459816188034584,
   'owner': '11111111111111111111111111111111',
   'rentEpoch': 90}},
 'id': 1}
